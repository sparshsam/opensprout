# OpenSprout — Windows Packaging Guide

**Version:** 0.9.14  
**Last updated:** 2026-06-25  
**Target:** Microsoft Store (Windows 10/11 PWA)

---

## Overview

OpenSprout is delivered as a Progressive Web App (PWA) on Windows. The PWA is generated by Next.js static export and hosted on Vercel. This guide covers packaging the PWA for Microsoft Store submission using PWABuilder (web tool) or manual MSIX generation.

## Prerequisites

- PWA deployed to `https://sprout.kovina.org` (or production domain)
- Microsoft Partner Center account ([register](https://partner.microsoft.com/dashboard))
- Code signing certificate (see Section 4)
- Store screenshots (see `docs/assets/store-screenshots/`)
- Privacy policy URL: `https://sprout.kovina.org/privacy`
- Terms of service URL: `https://sprout.kovina.org/terms`
- Support URL: `https://sprout.kovina.org/support`

---

## Option A: PWABuilder Web Tool (Recommended)

[PWABuilder](https://pwabuilder.com) generates signed MSIX packages directly from your PWA URL.

### Steps

1. Go to https://pwabuilder.com
2. Enter `https://sprout.kovina.org`
3. Click **Start**
4. Click **Package for Store** → **Windows**
5. Review the generated package settings:
   - **Publisher Display Name:** Sparsh Sam
   - **App Name:** OpenSprout
   - **Version:** Match `0.9.14`
6. Click **Generate**
7. Download the `.msixbundle` file
8. Sign with your code signing certificate (see Section 4)

### Edge DevTools Alternative

Edge DevTools can also generate MSIX packages:

1. Open `https://sprout.kovina.org` in Edge
2. Open DevTools (F12) → **Application** → **Manifest**
3. Look for **"Generate MSIX package"** or similar prompt
4. Fill in publisher info, select signing cert
5. Build package

## Option B: Manual MSIX Generation (PowerShell)

Run the following PowerShell script on a Windows 10/11 machine with the Windows App SDK installed.

### Prerequisites for Manual Build

- Windows 10 (build 1809+) or Windows 11
- [Windows App SDK](https://learn.microsoft.com/windows/apps/windows-app-sdk/) installed
- Visual Studio with "Universal Windows Platform development" workload
- Code signing certificate installed in the local certificate store

### PowerShell Script

```powershell
# scripts/package-windows.ps1
# Run from PowerShell as Administrator on Windows 10/11

param(
    [string]$PwaUrl = "https://sprout.kovina.org",
    [string]$PackageVersion = "0.9.14.0",
    [string]$Publisher = "CN=Sparsh Sam"
)

Write-Host "=== OpenSprout Windows MSIX Generator ===" -ForegroundColor Green

# Validate
if (-not (Get-Command "MakeAppx" -ErrorAction SilentlyContinue)) {
    Write-Host "MakeAppx not found. Install Windows App SDK." -ForegroundColor Red
    exit 1
}

# Create working directory
$workDir = ".\opensprout-msix"
New-Item -ItemType Directory -Force -Path $workDir | Out-Null

# Generate AppxManifest
@"
<?xml version="1.0" encoding="utf-8"?>
<Package
  xmlns="http://schemas.microsoft.com/appx/manifest/foundation/windows10"
  xmlns:uap="http://schemas.microsoft.com/appx/manifest/uap/windows10"
  xmlns:uap3="http://schemas.microsoft.com/appx/manifest/uap/windows10/3"
  xmlns:rescap="http://schemas.microsoft.com/appx/manifest/foundation/windows10/restrictedcapabilities"
  IgnorableNamespaces="uap uap3 rescap">

  <Identity Name="OpenSprout"
            Publisher="$Publisher"
            Version="$PackageVersion"
            ProcessorArchitecture="neutral" />

  <Properties>
    <DisplayName>OpenSprout</DisplayName>
    <PublisherDisplayName>Sparsh Sam</PublisherDisplayName>
    <Description>Free and open-source plant care tracking. Track watering, fertilizing, and care schedules for all your plants.</Description>
    <Logo>assets\icon-512.png</Logo>
  </Properties>

  <Resources>
    <Resource Language="en-us" />
  </Resources>

  <Dependencies>
    <TargetDeviceFamily Name="Windows.Universal" MinVersion="10.0.17763.0" MaxVersionTested="10.0.22621.0" />
    <TargetDeviceFamily Name="Windows.Desktop" MinVersion="10.0.17763.0" MaxVersionTested="10.0.22621.0" />
  </Dependencies>

  <Applications>
    <Application Id="App" Executable="$target$" EntryPoint="$target$">
      <uap:VisualElements
        DisplayName="OpenSprout"
        Square150x150Logo="assets\icon-192.png"
        Square44x44Logo="assets\icon-64x64.png"
        Description="Free and open-source plant care tracking."
        BackgroundColor="#16784f">
        <uap:DefaultTile Wide310x150Logo="assets\icon-512.png" Square71x71Logo="assets\icon-128x128.png" Square310x310Logo="assets\icon-512.png" />
        <uap:SplashScreen Image="assets\splash.png" />
      </uap:VisualElements>
    </Application>
  </Applications>

  <Capabilities>
    <rescap:Capability Name="runFullTrust" />
    <Capability Name="internetClient" />
  </Capabilities>
</Package>
"@ | Out-File -FilePath "$workDir\AppxManifest.xml" -Encoding utf8

# Copy icons from the opensprout project
$iconDir = "$workDir\assets"
New-Item -ItemType Directory -Force -Path $iconDir | Out-Null
Copy-Item "apps/web/public/icons\icon-512.png" "$iconDir\icon-512.png"
Copy-Item "apps/web/public/icons\icon-192.png" "$iconDir\icon-192.png"
Copy-Item "apps/web/public/icons\icon-128x128.png" "$iconDir\icon-128x128.png"
Copy-Item "apps/web/public/icons\icon-64x64.png" "$iconDir\icon-64x64.png"

# Build the MSIX
MakeAppx pack /d "$workDir" /p "opensprout-$PackageVersion.msix" /l

Write-Host "MSIX generated: opensprout-$PackageVersion.msix" -ForegroundColor Green
Write-Host "Next: Sign with code signing certificate." -ForegroundColor Yellow
```

## Section 3: Application Identity

### Package Identity Fields

| Field | Value | Notes |
|-------|-------|-------|
| Package Name | `OpenSprout` | Must be unique in Partner Center |
| Publisher | `CN=Sparsh Sam` | Must match code signing cert |
| Version | `0.9.14.0` | Quad-part (`major.minor.build.revision`) |
| Architecture | `neutral` | PWA runs on any arch |

### Changing Identity for Store

When submitting to Microsoft Store, the Publisher will be assigned by Partner Center. Update the manifest accordingly:

```xml
<Identity Name="StoreAssignedName"
          Publisher="CN=...AssignedByPartnerCenter"
          Version="0.9.14.0" />
```

## Section 4: Code Signing

### Option 1: Self-signed (sideloading only)

```powershell
New-SelfSignedCertificate -Type Custom -Subject "CN=Sparsh Sam" `
  -KeyUsage DigitalSignature -TextExtension @("2.5.29.37={text}1.3.6.1.5.5.7.3.3") `
  -CertStoreLocation "Cert:\CurrentUser\My"
```

### Option 2: Public CA (Store submission)

Purchase a code signing certificate from:
- **DigiCert** — Microsoft Authenticode
- **Sectigo** — Code Signing
- **GlobalSign** — Code Signing

### Sign the MSIX

```powershell
signtool sign /fd SHA256 /a /f "path\to\certificate.pfx" /p "password" `
  /v "opensprout-0.9.14.0.msix"
```

## Section 5: Microsoft Store Submission Checklist

### Required for Submission

- [ ] Microsoft Partner Center account ($19 one-time fee)
- [ ] Code signing certificate (public CA for Store submission)
- [ ] Signed MSIX package
- [ ] App description (short + full) — see `docs/store-listing.md`
- [ ] Privacy policy URL (`https://sprout.kovina.org/privacy`)
- [ ] App logo (300×300 PNG) — use `public/icons/icon-512.png`
- [ ] Screenshots (2+ required) — `docs/assets/store-screenshots/`
- [ ] Feature graphic (1350×540) — needs design tool
- [ ] Category: Lifestyle
- [ ] Age rating: Everyone
- [ ] Pricing: Free (no paid downloads)

### Submission Workflow

1. Sign in to [Microsoft Partner Center](https://partner.microsoft.com/dashboard)
2. Go to **Apps and Games** → **New product** → **Add new app**
3. Enter app name: **OpenSprout**
4. Fill in all store listing fields
5. Upload screenshots and logo
6. Upload signed MSIX package
7. Complete certification questionnaire
8. Submit for certification
